# Troubleshooting Guide

This guide collects common errors, root causes, and solutions encountered when configuring, running, and deploying the Next.js SaaS prototype.

---

## 1. Stripe Webhook Signature Verification Failures

### Symptom
- Console logs show: `[Webhook] Signature verification failed`
- Next.js API route `/api/webhooks/stripe` returns status `400 Bad Request` or `Invalid signature`.
- Stripe CLI reports failure logs.

### Root Cause
The `STRIPE_WEBHOOK_SECRET` (in production) or `STRIPE_CLI_WEBHOOK_SECRET` (locally) does not match the signing key generated by Stripe.

### Solution
1. **Local Development**:
   - Restarting `stripe listen` can generate a *new* local signing secret.
   - Look at your `stripe listen` console output. It will state:
     `Your webhook signing secret is: whsec_abc123...`
   - Copy this value exactly and paste it into `.env.local` as `STRIPE_CLI_WEBHOOK_SECRET`.
   - **Crucial**: Restart your Next.js server (`npm run dev`) so it reads the new environment variables.
2. **Production/Staging**:
   - Go to your Stripe Dashboard (Developers -> Webhooks), select the webhook endpoint, and click **Reveal Signing Secret**.
   - Ensure `STRIPE_WEBHOOK_SECRET` in Vercel settings matches this key exactly.

## 2. Webhook Database Upsert Fails

### Symptom
- Stripe CLI reports `checkout.session.completed` [500 Internal Server Error].
- Console logs show: `[Webhook] Failed to upsert subscription: ...` or database permissions errors.

### Root Causes
- **Missing Service Role Key**: The webhook needs to insert database records for users who aren't logged in. If RLS is enabled and the database client doesn't use the `service_role` key, the write is blocked.
- **Table Missing**: The `subscriptions` table was not initialized on Supabase.

### Solution
1. Check that `SUPABASE_SERVICE_ROLE_KEY` is defined in `.env.local` or Vercel settings and matches the **service_role** key (starts with `ey...` in your API settings), **not** the anon key.
2. Ensure you initialized your database client via `createServiceClient()` in the webhook route, which uses the service role bypass.
3. Confirm you ran the migrations in Supabase by running `001_subscriptions.sql` in the SQL Editor.

## 3. Redirect Loop Between Dashboard and Login

### Symptom
- Accessing `http://localhost:3000/dashboard` triggers infinite browser page redirects, or Chrome displays `ERR_TOO_MANY_REDIRECTS`.

### Root Cause
- The Supabase Auth session cannot be resolved on the server side because cookies are not being sent or read correctly.
- Alternatively, there is a mismatch between the environment project URL and the actual URL of the page.

### Solution
1. Clear your browser cookies and site data for `localhost:3000`, then reload.
2. Verify that `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are correct in `.env.local`.
3. In [lib/supabase/middleware.ts](file:///e:/76EAST/saas-product/lib/supabase/middleware.ts), make sure `request.cookies.getAll()` and `supabaseResponse.cookies.set()` logic is correct.
4. Check that your Next.js server is running on the default port matching `NEXT_PUBLIC_BASE_URL` (usually `http://localhost:3000`).

## 4. Stripe Checkout Session Fails to Create

### Symptom
- Clicking **Subscribe** leads to a 500 error in the console.
- Log error: `[Checkout Error] StripeInvalidRequestError: No such price: price_...`

### Root Cause
The value of `STRIPE_PRICE_ID` in `.env.local` does not exist in the connected Stripe account, or you are querying a live-mode price inside a test-mode API key configuration (or vice versa).

### Solution
1. Open the Stripe Dashboard (ensure **Test Mode** toggle is set correctly).
2. Go to **Products** and click your product. Look at the API ID under **Pricing**.
3. It must start with `price_...` (e.g. `price_1Pabc...`).
4. Update `STRIPE_PRICE_ID` in `.env.local` to match.
5. Restart your Next.js server.
